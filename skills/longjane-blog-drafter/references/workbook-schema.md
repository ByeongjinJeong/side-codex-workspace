# Workbook Schema

Workbook file: `longjane_blog_briefing.xlsx`.

If only `longjane_blog_briefing.xlsm` exists, the harness may read it as a fallback, but the preferred source file is `.xlsx` unless real Excel macros are required.

Project organization:

- Visit/photo-based content is stored under `후기성/` and uses `후기성/longjane_blog_briefing.xlsx`.
- Information-roundup content is stored under `정보성/` and uses `정보성/longjane_info_briefing.xlsx`.
- The outer `longjane_blog_auto/` folder is an umbrella folder only. Do not mix briefing rows, drafts, photos, or cardnews across the two tracks.

Supported topic sheets:

- `숙소`
- `맛집·카페`
- `관광지`
- `교통`

Other sheets:

- `발행 현황`: human-facing status summary, generated from category sheets where possible; do not treat it as the primary source of article briefing data.
- `키워드 분석`: keyword candidates, header row 2.

Topic sheets use header row 2. Row 3 usually contains examples and should not be treated as data.

## Common Columns

All topic sheets include:

- `ID`
- `방문 날짜`
- `여행지`
- `상태`
- `자유 메모`
- `초안 파일명`

Do not ask the user to fill `국내/해외`, `목표 키워드`, `블로그탭 노출`, or `인플루언서 점유` in topic sheets. The harness/Codex workflow infers domestic vs. overseas from `여행지`, generates and checks keyword candidates, and writes keyword/SERP evidence to supporting outputs.

Use `ID` as the article key. Prefer local `photos/{ID}/` even if `사진 폴더ID` differs. If the visible ID is generated by an Excel formula and no cached formula value exists, synthesize the ID from the category prefix and row number, e.g. row 4 on `숙소` is `STAY-001`.

Header colors in the workbook:

- Yellow: required manual input.
- Blue: optional manual input.
- Green: automatic/formula output.

## Information-Roundup Workbook

Information-centered posts use a separate workbook: `정보성/longjane_info_briefing.xlsx`.

Recommended sheet:

- `정보성`

Recommended columns:

- `ID`: use `INFO-001`, `INFO-002`, etc.
- `여행지`
- `정보 주제`: weather, season, museum list, transit pass, itinerary, budget, packing, visa/regulation, booking, comparison, safety, or shopping guide.
- `상태`
- `우선순위`
- `메인 키워드`
- `서브 키워드`
- `핵심 질문`
- `대상 독자`
- `포함할 항목`
- `제외할 항목`
- `기준 시점`
- `추천 발행 시점`
- `트렌드 근거`
- `선정 이유`
- `카드뉴스 아이디어`
- `참고/출처 메모`
- `자유 메모`
- `초안 파일명`

Use the same status flow as visit-review rows. Keep source research notes under `정보성/sources/{ID}/`, draft files under `정보성/drafts/{ID}/`, and cardnews files under `정보성/cardnews/{ID}/` or `정보성/drafts/{ID}/cardnews/`.

For topic discovery, it is acceptable to create rows in `대기중` or `제안` status before the user approves drafting. Do not mark a proposed topic as `작성 시작` until it is selected for actual content production.

## Status Flow

Use this status order:

1. `대기중`: The user has not assigned the draft to Codex yet.
2. `작성 시작`: User-selected trigger for Codex automation.
3. `초안 작성 중`: Codex has started the draft pipeline; use this to recover interrupted work.
4. `초안 완료`: Codex has completed the markdown draft, photo plan, ordered photos, and Word render.
5. `발행 완료`: The user manually published the post.

Do not assume all non-empty rows are ready. For batch mode, treat `작성 시작` as ready. Ask before resuming rows already marked `초안 작성 중`.
